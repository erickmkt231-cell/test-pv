🏆 Atualização: Ranking de Vendedores + 📱 Aplicativo

Parte 1 — Atualizar as permissões no Supabase (obrigatório para o ranking)

Antes, cada vendedor só enxergava os próprios dados. Agora todos os vendedores logados podem ver os dados uns dos outros (para o ranking), mas cada um só pode alterar os seus.

1No Supabase, abra o SQL Editor → New query, cole e clique em Run: 
drop policy if exists "usuario_acessa_so_seus_dados" on public.app_state;

create policy "vendedores_leem_tudo"
  on public.app_state for select
  to authenticated using (true);

create policy "insere_so_o_proprio"
  on public.app_state for insert
  to authenticated with check (auth.uid() = user_id);

create policy "atualiza_so_o_proprio"
  on public.app_state for update
  to authenticated using (auth.uid() = user_id)
  with check (auth.uid() = user_id);

create policy "apaga_so_o_proprio"
  on public.app_state for delete
  to authenticated using (auth.uid() = user_id);
2Se ainda não fez: Authentication → Sign In / Providers → Email → desative "Confirm email".
🔒 Sobre privacidade: o ranking é público entre os vendedores logados — quem não tem conta não vê nada. É o formato mais seguro para dados financeiros.

Parte 2 — Subir os novos arquivos no GitHub

1No seu repositório, use Add file → Upload files e envie todos estes arquivos juntos (eles substituem os antigos):
index.html · manifest.json · sw.js · icon-192.png · icon-512.png · icon-maskable-512.png
2Commit changes e aguarde 1–2 minutos. Abra o site e confira: deve aparecer a aba 🏆 Ranking de Vendedores no menu.
Cada vendedor cria a própria conta na aba Criar conta (com o nome que aparece no ranking). O nome também pode ser alterado depois em Configurações.

Parte 3 — Instalar como aplicativo (sem APK)

Com esta atualização o sistema virou um PWA — um aplicativo instalável direto do navegador:

Android (Chrome): abra o site → menu ⋮ → "Adicionar à tela inicial" ou "Instalar aplicativo". Vira um app com ícone, tela cheia, sem barra de navegador.
iPhone (Safari): abra o site → botão Compartilhar → "Adicionar à Tela de Início".

Ele abre instantâneo, funciona offline (os dados sincronizam quando voltar a internet) e se atualiza sozinho quando você publicar uma versão nova no GitHub.

Parte 4 — Gerar o APK de verdade (para compartilhar o arquivo)

Para ter um arquivo .apk que você pode mandar no WhatsApp e instalar em qualquer Android:

1Acesse pwabuilder.com e cole a URL do seu site (a do GitHub Pages). Clique em Start.
2Ele analisa seu PWA (manifest e service worker já estão prontos ✓). Clique em Package for Stores → Android.
3Baixe o pacote. Dentro do .zip vem o APK pronto para instalar (use o arquivo "signed" para uso próprio).
4Envie o APK para os vendedores. No Android, ao instalar, é preciso permitir "instalar de fontes desconhecidas" (normal para apps fora da Play Store).
💡 O APK é uma "casca" que abre seu site — ou seja, quando você atualizar o sistema no GitHub, todos os apps atualizam sozinhos, sem gerar novo APK.

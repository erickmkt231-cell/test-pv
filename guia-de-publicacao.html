🐟 AquaGest — Guia de Publicação Gratuita

Este guia coloca seu sistema no ar de graça usando GitHub Pages (hospedagem do site) e Supabase (banco de dados + login). Leva uns 15 minutos, sem programar nada.

Parte 1 — Publicar o site no GitHub Pages

1Crie uma conta gratuita em github.com (se ainda não tiver).
2Clique no + (canto superior direito) → New repository. Nome sugerido: aquagest. Marque Public e clique em Create repository.
3Na página do repositório, clique em uploading an existing file (ou Add file → Upload files) e envie o arquivo index.html que recebeu junto com este guia. Clique em Commit changes.
4Vá em Settings → Pages (menu lateral). Em "Branch", escolha main e pasta / (root), depois Save.
5Aguarde 1–2 minutos. Seu sistema estará no ar em:
https://SEU-USUARIO.github.io/aquagest/
📱 No celular: abra esse endereço no navegador e use "Adicionar à tela inicial" — vira um ícone como um aplicativo.

Parte 2 — Criar o banco de dados no Supabase

1Crie uma conta gratuita em supabase.com e clique em New project. Dê um nome (ex.: aquagest), crie uma senha de banco qualquer (guarde-a) e escolha a região South America (São Paulo).
2Quando o projeto abrir, vá em SQL Editor (menu lateral), cole o código abaixo e clique em Run: 
create table if not exists public.app_state (
  user_id uuid primary key references auth.users(id) on delete cascade,
  data jsonb not null,
  updated_at timestamptz default now()
);

alter table public.app_state enable row level security;

create policy "usuario_acessa_so_seus_dados"
  on public.app_state
  for all
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);
Deve aparecer "Success". Isso cria a tabela onde seus dados ficam guardados, protegida para que só você acesse os seus.
3Vá em Authentication → Sign In / Providers → Email e desative a opção "Confirm email". Assim você cria sua conta e entra na hora, sem precisar confirmar e-mail.
4Vá em Project Settings → API (ou "Data API") e copie dois valores:
• Project URL (algo como https://abcd1234.supabase.co)
• anon public key (um código longo começando com eyJ...)

Parte 3 — Conectar o sistema à nuvem

1Abra seu sistema (o endereço do GitHub Pages). Na tela inicial, toque em ☁️ Conectar à nuvem (Supabase).
2Cole a Project URL e a chave anon e salve.
3Na tela de login, digite seu e-mail e uma senha (mínimo 6 caracteres) e toque em Criar conta nova. Pronto — você está logado e tudo o que registrar sincroniza automaticamente.
4Para usar em outro aparelho: abra o mesmo endereço, conecte o mesmo Supabase (mesma URL e chave) e entre com o mesmo e-mail e senha. Seus dados aparecem lá.
⚠️ Importante: a chave "anon" é pública e segura de usar no site (a proteção real é o login + a regra criada no Passo 2). Nunca use a chave service_role no site. E exporte um backup de vez em quando em Configurações → Exportar backup.

Problemas comuns

“Falha de conexão com o Supabase” — confira se copiou a URL completa (com https://) e a chave anon inteira.
“Confirme pelo link enviado ao seu e-mail” — você esqueceu de desativar o "Confirm email" (Parte 2, passo 3). Desative e entre de novo, ou clique no link do e-mail recebido.
Site não abre após ativar o Pages — aguarde 2 minutos e confira se o arquivo se chama exatamente index.html.
